Autonomous Robot

A University Project

In the third year of my mechatronics degree, teams were presented with the problem of creating an autonomous robot to navigate a room. My team decided to use two control boards, to achieve two goals. The first board, a Nucleo f303k8, was programmed to communicate with the hardware directly, using a custom pure pursuit algorithm and encoder/gyroscope odometery to position the robot. The second board, a Raspberry Pi, was programmed with a custom Real Time Operating System to receive data from a LiDAR and user (over Wi-Fi) to inform its overall path. The mother board (Raspberry Pi), is in constant communication with daughter board to give movement instructions and verify position data with the absolute position, calculated from the LiDAR. This project also allowed me to create Windows and Android applications (C#), for the user to see the status of the robot and give instructions.

Coil Winding System

University Product Design

Coil Winding

University Product Design

In the second year of my mechatronics degree, teams were tasked with creating a coil winding system from scratch. My team decided to create an open bed model, to allow quick and easy prototyping or component replacement. This project required me to create a coil winding and serial communication program in C++ for use on a STM32 FR446RE microcontroller. By using stepper motors and a tensioning system, a simple velocity PID could be implemented to maintain the speed between the input and output coils. In addition, I created a windows application using C# for the user to interface with, focusing on an intuitive and easy layout. These settings would then be sent via UART serial to the microcontroller and applied.

Kiwifruit Recognition for Zespri

Internship Project

Kiwifruit Recognition for Zespri

Internship Project

During the summer of 2020 I was able to work with Zespri to create a kiwifruit recognition system. Initially it was thought that artificial intelligence would be the best solution, this proved to be accurate in ideal situations, but failed under low light. A colour analysis algorithm would object permanence was then tested to give better results in all cases tested. Finally, I created some C++ code to turn the Time-of-Flight camera data into a 3d model, which could then be analysed using a custom plugin for blender, which was then returned and matched back to the regular camera data for exact edge detection. This is then presented to the user in a C# application.
